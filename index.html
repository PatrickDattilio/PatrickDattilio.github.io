Hello! I think we should talk about what I can do for Basecamp. Thanks for taking the time to look over a few ideas. Lets get to it!

FAB Please
The menu must flow

This dropdown menu is the perfect candidate for a Floating Action Button menu, similar to Gmail/Inbox's. It's easier to reach as a FAB in the bottom right corner and saves us valuable Toolbar space. Now that we've moved the + menu to a FAB, we have plenty of space to move these overflow menu actions right into our toolbar. Consider changing these from showAsAction="never" to "ifRoom", or even "ifRoom|withText" to show both an icon and it's associated title.

Lets talk about text baby
Release the menu

The message screen has four actions hidden in an overflow menu that could easily be represented as icons if there is room. This becomes even more apparent in landscape, where we could easily fit all the actions with text.

ifRoom will show

The calendar offers another example where changing the menu's "showAsAction" would prevent the refresh button from being hidden without need.

I would replace this dialog with a simple in-place dropdown menu, this is pre-honeycomb spinner design.

Let there be dropdowns
Todo: Inline Todo editing

Adding an empty EditText at the bottom of the Todo listwould allow quick in-line additions, a pattern I have seen in other todo/task lists. Long clicking on an item in the list to enable in-line edits would feel very intuative.

Clicking on the image in a discussion opens up the image fullscreen, but I found it disorienting to have to click back three times to return this screen. We might consider reducing the artifical backstack here. I expected to be able to swipe between attached images, but we have to back out instead. Allowing us to swipe from a message to each of the fullscreen attachments reduces the complexity.

It's like I'm swiping a marker or something.
